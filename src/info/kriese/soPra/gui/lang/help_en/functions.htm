General

Here you can edit the data for a LOP in second geometric interpretation (m = 2, n arbitrary). There is a minimum of three variables, due to the fact that LOP with two variables are better solved in the first geometric interpretation (m arbitrary, n = 2) or by a simple linear algebraic equation. The arrays of the table have to be filled with the coefficient of the x(i) of objective function (OF) and side conditions (SC). Variables who are not present in the objective function or in one of the side conditions have the coefficient zero. Minimizing or maximizing the LOP is chosen by a pulldown-menu. This point of the menu can be called at every stage of editing a LOP.


 

  Add variable

Here you can add additional variables (up to 10variables).


 

  Remove variable

Here you can remove variables. With every use of the button the column with the highest index will be deleted.


 

  Reset values

With this button all changes that have not been taken over by using the "Accept values" button will be deleted. The input window will be set back to the state of the last changes that have been accepted/saved. PAY ATTENTION: This step can not be reverted!


 

  Delete values

The whole contents of the table will be deleted. All coefficients will be reset to zero and the number of variables will be reduced to three. PAY ATTENTION: This step can not be reverted! All unsaved data will be lost!


 

  Accept values

With this step the edited data will be applied to the actual LOP. The LOP will be solved automatically with the given data. PAY ATTENTION: As long as there are invalid data within one of the arrays (background of the array is red) an error message will appear, and the data will not be applied to the LOP! Even yellow marked arrays have to be left by using the Return- or one of the arrow-keys. Otherwise the values of these arrays will not be applied!


 

  Validate solution

With this button the previously calculated and entered solution will be validated. If the LOP has no solution, "ne" (for "not existing") has to be entered as value of the solution. For unlimited LOP's "u" hast to be entered as value of the solution (both without the ""). The identified cases for solution, solution set and target function have to be entered by selecting the appropriate buttons.
